# 法护星 (FaHuXing)

**基层纠纷智慧维权 AI 助手**

一款基于人工智能技术的基层法律辅助服务平台，为普通用户提供从法律咨询、维权规划、证据收集到文书生成的一站式维权工具。无需专业法律知识，即可获取专业的法律建议和可执行的维权方案。

---

## 项目简介

法护星致力于降低基层群众获取法律服务的门槛，通过 AI 技术将复杂的法律流程简化为简单易懂的操作步骤。无论是劳动纠纷、房屋租赁、消费维权还是邻里纠纷，法护星都能提供及时、专业、可执行的法律帮助。

## 核心功能

### ⚖️ 法律咨询
描述纠纷事实，AI 自动分析案情并提供专业法律建议。系统会自动保存咨询记录并生成唯一 ID，供后续功能调用。

### 🧭 维权清单
基于纠纷描述，自动生成按时间轴划分的维权行动方案，包括：
- **24 小时内**：紧急措施与证据保全
- **7 天内**：短期维权行动
- **30 天内**：中期规划与跟进
- **风险提醒**：注意事项与风险提示

### 🗂️ 证据清单
智能生成证据材料清单，以表格形式呈现：
- 证据类别与具体材料
- 获取方式与注意事项
- 证明目的说明
- 证据保全建议

### 📝 文书生成
支持两种生成模式：

**AI 智能模式（元器）**
基于咨询记录，调用腾讯元器 API 智能分析案情，自动生成起诉状的结构化字段
（诉讼请求、事实与理由、法律依据、证据清单），并自动填入表单供用户编辑。

**表单填写模式**
- **手动填写**：手动填写原告/被告信息、诉讼请求、事实与理由等
- **智能填充**：点击"智能填充"按钮，元器自动根据法律咨询内容填充表单字段
- 生成后可按标准格式输出起诉状

### 🧮 维权计算器
内置实用计算工具：
- **加班费计算器**：支持工作日、休息日、法定节假日三种加班类型
- **经济补偿金计算器**：根据月薪和工作年限快速计算

## 快速启动

### 环境要求
- Python 3.10+（推荐 conda 环境）
- Node.js 18+
- PostgreSQL 14+ 或 Docker Desktop

### 方式一：Docker Compose（推荐）

```bash
# 1. 复制并编辑环境变量
cp backend/.env.example backend/.env
# 填入你的 API Key 和数据库配置

# 2. 一键启动所有服务
docker compose up -d

# 3. 访问服务
# 前端: http://localhost:3000
# 后端: http://localhost:8088
# API 文档: http://localhost:8088/docs
```

**注意**：现在项目支持一键启动，数据库的 vector 插件会自动激活，无需手动操作。

### 方式二：本地开发

#### 后端启动
```bash
# 激活 conda 环境   PS： 如果第一次运行，请你先创建一个环境，然后再激活
conda activate fahuxing_env

# 进入后端目录
cd backend

# 安装依赖
pip install -r requirements.txt

# 启动后端服务
uvicorn main:app --reload --host 127.0.0.1 --port 8088
```

#### 前端启动
```bash
# 新开一个终端，进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

启动后访问 http://localhost:3000

## 使用指南

### 第一步：法律咨询
1. 选择"法律咨询"功能
2. 输入纠纷事实描述（尽量详细）
3. 点击"获取法律建议"
4. 系统返回 AI 分析结果并生成咨询记录 ID

### 第二步：维权/证据清单
1. 切换到"维权清单"或"证据清单"
2. 系统基于纠纷描述自动生成方案
3. 查看详细的时间轴规划或证据材料表格

### 第三步：文书生成
**AI 智能模式：**
1. 切换到"文书生成"
2. 输入咨询记录 ID（或点击使用最近咨询 ID）
3. 点击生成，AI 自动分析案情生成起诉状

**表单填写模式：**
1. 切换到"文书生成"
2. 选择"表单填写模式"
3. 依次填写原告信息、被告信息、诉讼请求、事实与理由
4. 点击生成，系统按标准格式输出起诉状

### 第四步：金额估算
1. 切换到"计算器"功能
2. 选择加班费或经济补偿金计算器
3. 输入月薪、工作时长或工作年限
4. 系统自动计算结果

## 文书模板说明

项目内置标准民事起诉状模板，包含以下标准字段：
- 原告信息（姓名、性别、出生年月、住址、联系方式）
- 被告信息（姓名/单位、地址、联系方式）
- 诉讼请求（列表形式）
- 事实与理由（详细描述）
- 法律依据（可选）
- 类案参考（AI 自动生成）
- 证据目录（可选）
- 管辖法院
- 具状人与日期

模板文件位置：`backend/templates/lawsuit_standard.j2`

## 常见问题

### Q: 服务启动后页面无法访问？
A: 请检查：
1. 后端是否启动成功（查看终端日志）
2. 前端代理配置是否正确（next.config.mjs）
3. 端口是否被占用

### Q: 文书生成返回错误？
A: 请检查：
1. 咨询记录 ID 是否正确
2. 环境变量是否配置完整
3. API Key 是否有效

### Q: 端口冲突如何解决？
A: 修改 next.config.mjs 和 .env.local 中的端口配置，同时修改后端启动端口。
